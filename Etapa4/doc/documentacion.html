2. Arquitectura MVC

El proyecto sigue el patrón Modelo–Vista–Controlador:

CapaClasesResponsabilidad
Modelo Equipo, Cellular, EloTelTag, Tablet, ETNube, Territory Almacenan datos y lógica de negocio. La posición se guarda en DoubleProperty para permitir enlace automático con las vistas.
Vista CellularView, EloTelTagView, TabletView, TerritoryView, FindMyWindow Renderizan cada dispositivo usando nodos JavaFX. Se actualizan automáticamente mediante bind() sin necesidad de llamadas manuales.
Controlador Stage4, Territory.moveAll() Reacciona a eventos (Play/Pause, clic en dispositivo) y coordina timelines de animación.

3. Descripción de Clases

Equipo

Clase base para todos los dispositivos. Usa DoubleProperty de JavaFX para la posición, lo que permite enlace automático con las vistas. El movimiento sigue un modelo polar: en cada paso de tiempo el ángulo varía aleatoriamente en ±dtheta y la posición avanza r·dt píxeles. Rebote elástico al alcanzar los bordes.

Cellular

Extiende Equipo. Mantiene referencia a ETNube. Reporta su propia posición cada 4 segundos (reportLocation()) y la posición aproximada de tags/tablets cercanos (reportTagLocation(tag)).

EloTelTag

Extiende Equipo. Añade un nombre propio (p.ej. "maleta"). Detecta celulares en un radio de 50 px (isWithinRange()). Busca celulares cada 4 segundos.

Tablet

Extiende EloTelTag con nombre fijo "tablet". Busca celulares cada 5 segundos. Se hereda todo el comportamiento de rastreo.

Territory

Controlador de la simulación. Contiene todos los equipos en un ArrayList<Equipo> genérico. moveAll(dt) delega el movimiento a cada equipo. reportTagLocations() y reportTabletLocations() iteran los equipos y detectan celulares cercanos para reportar a la nube.

ETNube

Servicio de nube (thread-safe con synchronized). Almacena la última posición conocida de cada dispositivo por propietario. getOwnerRecords(owner) devuelve todos los bienes de una persona para mostrar en FindMy.

TerritoryView

Extiende ScrollPane. Contiene un StackPane con la imagen de fondo y un Pane superpuesto donde se añaden las vistas de equipos. WIDTH y HEIGHT estáticos definen los límites del territorio para el rebote.

CellularView / EloTelTagView / TabletView

Vistas de cada dispositivo que extienden Group. Las propiedades de posición de las formas JavaFX se enlazan con bind() al modelo, por lo que se actualizan sin código adicional. EloTelTagView y TabletView añaden animación de radar y sonido en showRadar(). Todas incluyen menú contextual con "Find My".

FindMyWindow

Ventana flotante que muestra los bienes de una persona. Se auto-actualiza cada segundo mediante un Timeline interno. Se detiene limpiamente al cerrar la ventana.

SoundPlayer

Genera y reproduce un pitido corto (880 Hz, 120 ms) por síntesis PCM usando javax.sound.sampled. No requiere archivos externos.
